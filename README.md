## Key Features

- Incident reporting and tracking
- Real-time disaster notifications and alerts
- Volunteer registration and task assignment
- Resource and inventory management
- Shelter location and status management
- GIS-based mapping and visualization
- Role-based access control (civilian, volunteer, coordinator)
- Scalable microservices-based backend architecture

---

## System Architecture

The system follows a distributed architecture composed of:

- **Frontend Application** – User interface for civilians, volunteers, and coordinators
- **Backend API** – Core business logic and data processing layer
- **Database Layer** – Persistent storage (PostgreSQL)
- **Cache & Messaging** – Redis for caching and messaging
- **Object Storage** – MinIO for storing unstructured data (images, documents)
- **Deployment** – Containerized deployment using Docker/Kubernetes
